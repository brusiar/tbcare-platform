🏗️ ARQUITETURA IMPLEMENTADA
───────────────────────────────────────────────────────────────────────────────

Padrão: Monolito Modular
  • Cada módulo é independente e pode ser extraído como microsserviço
  • Comunicação entre módulos apenas via camada service
  • Separação clara de responsabilidades

Camadas:
  Controller → Service → Repository → Domain

Multi-Tenant:
  • tenant_id presente em todas as entidades de negócio
  • Herança via TenantAwareEntity
  • Preparado para filtro automático (próxima etapa)

API REST:
  • Resposta padrão unificada (ApiResponse<T>)
  • Tratamento global de exceções
  • CORS configurado

🚀 COMO EXECUTAR
───────────────────────────────────────────────────────────────────────────────

Opção 1 - Docker (Recomendado):
  $ docker compose up --build

Opção 2 - Local:
  Terminal 1: $ docker compose up postgres
  Terminal 2: $ cd backend && mvn spring-boot:run
  Terminal 3: $ cd frontend && npm install && npm run dev

Verificar ambiente:
  $ ./verify.sh

Acessos:
  • Frontend: http://localhost:3003
  • Backend: http://localhost:8082/api
  • Health: http://localhost:8082/api/health
  • Postgres: localhost:5434
